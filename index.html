Introduction

Why this guide exists

What is this?

An interactive documentation site that teaches the CSS properties responsible for typography (how text looks) and layout (how boxes are placed). Every concept is paired with a live demo and the exact code that produced it.

Why typography matters

Text is the most common element on the web. The right font-size, line-height and spacing can make a page feel calm and readable — or cramped and exhausting. Good typography builds trust, improves comprehension and reinforces brand identity.

Why layout matters

Layout properties (display, position, float, z-index) decide where boxes go and how they react to each other. Mastering them is the difference between a fragile page and a responsive, robust one.

Who is it for?

Beginners learning CSS for the first time, and intermediate developers who want a quick visual refresher. No JavaScript, no build step — open index.html and read.

Learning objectives

By the end of this guide you will be able to:

  • Choose the correct font-* property for any text effect.
  • Control spacing, alignment and wrapping of paragraphs.
  • Style lists, tables and decorative content confidently.
  • Predict how display, position and float change the box model.
  • Decide between visibility:hidden, opacity:0 and display:none.
  • Apply the right cursor and accessibility cues for interactive elements.

1 · Typography

How text looks: faces, size, weight, spacing & decoration

typography

font-family

Definition. Sets the typeface (font) used for an element's text. A fallback list lets the browser pick the first available font.

Syntax

selector {
  font-family: "Primary Font", "Fallback Font", generic-family;
}

Accepted values

  • serif — fonts with finishing strokes (Times, Georgia)
  • sans-serif — clean, stroke-less fonts (Arial, Helvetica)
  • monospace — equal-width characters (Consolas, Courier)
  • cursive / fantasy — decorative families
  • Any named font family, in quotes if it has spaces

Visual demonstration

HTML
<p class="serif">The quick brown fox</p>
<p class="mono">The quick brown fox</p>
CSS
.serif { font-family: Georgia, "Times New Roman", serif; }
.mono  { font-family: Consolas, monospace; }

Rendered output

The quick brown fox jumps over the lazy dog.

The quick brown fox jumps over the lazy dog.

💡 Tips

Always end the list with a generic family so the page still works if no named font is installed.

🌐 Browser notes

The browser uses the first matching font per glyph, so mixed scripts can render in different faces.

✅ Common use cases

Body text, headings, code blocks, branding through a custom web font.

typography

font-size

Definition. Controls the height of the text characters.

Syntax

selector { font-size: <length> | <percentage> | <keyword>; }

Accepted values

  • px — absolute pixels (16px)
  • em / rem — relative to parent / root font size
  • % — relative to the inherited size
  • Keywords: xx-small … xx-large, smaller, larger

Visual demonstration

HTML
<p class="xs">Aa</p>
<p class="xl">Aa</p>
CSS
.xs { font-size: 12px; }
.xl { font-size: 44px; }

Rendered output

Small text sample

Large text sample

💡 Tips

Prefer rem for body text so it respects the user's browser font-size setting.

🌐 Browser notes

Default browser size is 16px = 1rem.

✅ Common use cases

Headings hierarchy, readable body copy, responsive scaling.

typography

font-style

Definition. Selects whether text is normal, italic or oblique (slanted).

Syntax

selector { font-style: normal | italic | oblique; }

Accepted values

  • normal
  • italic — uses the italic face
  • oblique — artificially slants normal face

Visual demonstration

HTML
<em>italic by default</em>
<i class="oblique">oblique</i>
CSS
.oblique { font-style: oblique; }

Rendered output

Normal paragraph text.

Italic paragraph text.

Oblique paragraph text.

💡 Tips

Use <em> for emphasis (semantic), <i> for purely visual italics.

🌐 Browser notes

If no italic face exists, browsers synthesise one from the normal face.

✅ Common use cases

Emphasis, captions, book titles, pull-quotes.

typography

font-weight

Definition. Sets how bold or light the text appears, from 100 (thin) to 900 (black).

Syntax

selector { font-weight: normal | bold | 100..900; }

Accepted values

  • Keywords: normal (400), bold (700)
  • lighter / bolder — relative to inherited
  • Numeric: 100 200 300 400 500 600 700 800 900

Visual demonstration

HTML
<p class="thin">Light</p>
<p class="black">Heavy</p>
CSS
.thin  { font-weight: 100; }
.black { font-weight: 900; }

Rendered output

Thin (100) text sample.

Bold (700) text sample.

Black (900) text sample.

💡 Tips

Intermediate weights (100–900) only work if the font file actually ships them.

🌐 Browser notes

System fonts like Arial often only have 400 and 700.

✅ Common use cases

Headings, emphasis, buttons, hierarchy without changing size.

Syntax

Accepted values

  • all-small-caps — also shrinks capitals (CSS3)

Visual demonstration

CSS
.smc { font-variant: small-caps; }

Rendered output

The quick brown fox

💡 Tips

Modern CSS3 also offers font-variant-numeric for tabular numbers.

🌐 Browser notes

If the font has no small-caps face, browsers synthesise it.

✅ Common use cases

Acronyms, labels, elegant headings.

typography

font-stretch

Definition. Selects a normal, condensed or expanded face of a font family.

Syntax

selector { font-stretch: ultra-condensed | condensed | normal | expanded | ultra-expanded; }

Accepted values

  • condensed · semi-condensed · normal
  • semi-expanded · expanded
  • ultra-condensed · ultra-expanded
  • Percentages e.g. 50%–200%

Visual demonstration

HTML
<p class="wide">Wide</p>
CSS
.wide { font-stretch: expanded; }

Rendered output

Condensed text sample.

Normal text sample.

Expanded text sample.

💡 Tips

Only affects fonts that actually ship condensed/expanded faces.

🌐 Browser notes

Falls back to normal when the requested face is unavailable.

✅ Common use cases

Fitting text into narrow columns, display headlines.

typography

line-height

Definition. Sets the vertical space between lines of text (leading).

Syntax

selector { line-height: normal | <number> | <length> | <percentage>; }

Accepted values

  • normal ≈ 1.2
  • Unitless number (recommended): 1.5
  • px, em, %

Visual demonstration

HTML
<p class="read">Two lines of<br>text here.</p>
CSS
.read { line-height: 1.5; }

Rendered output

Line one of the paragraph.
Line two of the paragraph.

Line one of the paragraph.
Line two of the paragraph.

💡 Tips

Use a unitless value so descendants inherit the multiplier, not a fixed size.

🌐 Browser notes

1.4–1.6 is the readability sweet spot for body text.

✅ Common use cases

Body paragraphs (larger), buttons & nav (tighter).

typography

letter-spacing

Definition. Increases or decreases the space between characters (tracking).

Syntax

selector { letter-spacing: normal | <length>; }

Accepted values

  • normal
  • Positive length (2px) widens
  • Negative length (-1px) tightens

Visual demonstration

HTML
<p class="spread">TRACK</p>
CSS
.spread { letter-spacing: 4px; }

Rendered output

Spacing demo text.

Spacing demo text.

💡 Tips

Wide tracking looks elegant on uppercase labels & headings.

🌐 Browser notes

The last letter also gets trailing space — account for it in tight layouts.

✅ Common use cases

Eyebrow labels, logos, all-caps headings.

typography

word-spacing

Definition. Sets the spacing between words, added on top of the normal space.

Syntax

selector { word-spacing: normal | <length>; }

Accepted values

  • normal
  • Positive length widens gaps
  • Negative length shrinks gaps

Visual demonstration

HTML
<p class="gappy">words apart</p>
CSS
.gappy { word-spacing: 12px; }

Rendered output

A few words placed apart here.

A few words placed apart here.

💡 Tips

Often combined with text-align: justify for even ragged edges.

🌐 Browser notes

Word boundaries follow language-specific rules.

✅ Common use cases

Justified text, artistic headlines, justified book layouts.

typography

text-align

Definition. Sets the horizontal alignment of inline content within a block.

Syntax

selector { text-align: left | right | center | justify; }

Accepted values

  • left · right · center
  • justify — stretches lines to full width
  • start / end — logical, follow direction

Visual demonstration

HTML
<p class="c">Centered</p>
CSS
.c { text-align: center; }

Rendered output

This paragraph is aligned to the left edge of its container.

This paragraph is centered within its container.

💡 Tips

For RTL languages use start/end instead of left/right.

🌐 Browser notes

✅ Common use cases

Centering titles, right-aligning numbers in tables, justified articles.

typography

text-decoration

Definition. Draws lines on text — underline, overline or strike-through — and can style them.

Syntax

selector { text-decoration: <line> <style> <color> | none; }

Accepted values

  • Line: underline · overline · line-through · none
  • Style: solid · dotted · dashed · wavy
  • Color: any CSS color

Visual demonstration

HTML
<a class="plain">no underline</a>
<span class="err">wavy</span>
CSS
.plain { text-decoration: none; }
.err   { text-decoration: underline wavy red; }

Rendered output

Text with an underline.

Text with a strike-through.

Text with a wavy red underline.

💡 Tips

Remove default underlines on links with text-decoration: none, but keep another cue for accessibility.

🌐 Browser notes

Multiple lines can be combined: underline overline.

✅ Common use cases

Links, errors (wavy), deleted prices (line-through).

typography

text-transform

Definition. Changes the capitalisation of text without altering the source HTML.

Syntax

selector { text-transform: none | capitalize | uppercase | lowercase; }

Accepted values

  • none
  • capitalize — first letter of each word
  • uppercase / lowercase
  • full-width — fixed-width square glyphs

Visual demonstration

HTML
<p class="up">hello world</p>
CSS
.up { text-transform: uppercase; }

Rendered output

hello world (unchanged)

hello world (uppercased)

hello world (capitalized)

💡 Tips

Great for consistent labels & badges without retyping content.

🌐 Browser notes

Language-specific rules (e.g. Turkish dotted-i) are respected.

✅ Common use cases

Buttons, tags, section eyebrows, displaying user input uniformly.

typography

text-indent

Definition. Indents the first line of a block of text.

Syntax

selector { text-indent: <length> | <percentage>; }

Accepted values

  • Length: 2em, 20px
  • Percentage of containing block width
  • Negative value for "outdent" (hanging indent)

Visual demonstration

HTML
<p class="indent">First line…</p>
CSS
.indent { text-indent: 2em; }

Rendered output

The first line of this paragraph is pushed in by two em units, while the wrapped lines return to the normal left edge.

💡 Tips

A negative text-indent plus padding makes a hanging indent (bibliographies).

🌐 Browser notes

Only affects the first formatted line, not every line.

✅ Common use cases

Book-style paragraphs, bibliographies, pull-quote styling.

typography

text-shadow

Definition. Adds one or more shadows behind the text glyphs.

Syntax

selector { text-shadow: <offset-x> <offset-y> <blur> <color>; }

Accepted values

  • Two offsets (required): 2px 2px
  • Optional blur radius
  • Optional color (last)
  • Multiple shadows comma-separated

Visual demonstration

HTML
<h1 class="glow">Shadow</h1>
CSS
.glow { text-shadow: 0 0 10px rgba(99,179,237,.8); }

Rendered output

Text without a shadow.

Text with a soft shadow.

Text with a glowing shadow.

💡 Tips

Use text-shadow to keep text legible over busy background images.

🌐 Browser notes

Shadows never affect layout — they can overflow the box.

✅ Common use cases

Hero headlines, neon/glow effects, depth on buttons.

typography

text-overflow

Definition. Decides how overflowing text is signalled when overflow is hidden and text does not wrap.

Syntax

selector {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: clip | ellipsis;
}

Accepted values

  • clip — cut at the edge
  • ellipsis — show …
  • Custom string (limited support)

Visual demonstration

HTML
<p class="truncate">Long text…</p>
CSS
.truncate {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

Rendered output

A very long piece of overflowing sample text inside a narrow box.

A very long piece of overflowing sample text inside a narrow box.

💡 Tips

The "truncate" trio above is the most reusable single-line pattern in CSS.

🌐 Browser notes

Requires overflow:hidden + white-space:nowrap to take effect.

✅ Common use cases

File names, email subjects, table cells, breadcrumbs.

typography

white-space

Definition. Controls how whitespace and line breaks inside an element are handled.

Syntax

selector { white-space: normal | nowrap | pre | pre-wrap | pre-line; }

Accepted values

  • normal — collapse spaces, wrap
  • nowrap — never wrap
  • pre — preserve exactly like <pre>
  • pre-wrap — preserve but still wrap
  • pre-line — collapse spaces, keep newlines

Visual demonstration

HTML
<p class="keep">one   two
three</p>
CSS
.keep { white-space: pre-wrap; }

Rendered output

one two three (collapsed to single spaces)

one two three (preserved exactly)

💡 Tips

pre-wrap is ideal for showing code or poetry without horizontal scroll.

🌐 Browser notes

Pair with overflow-wrap to handle very long unbreakable strings.

✅ Common use cases

Code blocks, badges, preserving user formatting.

typography

word-break

Definition. Controls whether the browser may break lines inside words to prevent overflow.

Syntax

selector { word-break: normal | break-all | keep-all; }

Accepted values

  • normal
  • break-all — break anywhere, even mid-word
  • keep-all — no breaks for CJK word boundaries

Visual demonstration

HTML
<p class="break">LongStringNoSpaces</p>
CSS
.break { word-break: break-all; }

Rendered output

SupercalifragilisticexpialidociousWordOverflowing

SupercalifragilisticexpialidociousWordBreakingMidWord

💡 Tips

Prefer overflow-wrap: break-word first — it only breaks when necessary.

🌐 Browser notes

break-all can split words awkwardly in Latin scripts.

✅ Common use cases

URLs, long tokens, narrow table columns.

typography

overflow-wrap

Definition. Allows the browser to break an otherwise unbreakable word only if it would overflow the container.

Syntax

selector { overflow-wrap: normal | break-word | anywhere; }

Accepted values

  • normal
  • break-word — break only when needed
  • anywhere — also affects min-content sizing

Visual demonstration

HTML
<p class="wrap">Hippopotomonstrosesquippedaliophobia</p>
CSS
.wrap { overflow-wrap: break-word; }

Rendered output

HippopotomonstrosesquippedaliophobiaLongUnbreakableString

HippopotomonstrosesquippedaliophobiaLongUnbreakableString

💡 Tips

Formerly called word-wrap; both names still work.

🌐 Browser notes

Friendlier than word-break: break-all for normal prose.

✅ Common use cases

User-generated content, comments, chat messages, URLs.

typography

vertical-align

Definition. Aligns inline (or table-cell) elements vertically relative to their parent or line box.

Syntax

selector { vertical-align: baseline | sub | super | top | middle | bottom | <length>; }

Accepted values

  • Keywords: baseline, top, middle, bottom, text-top, text-bottom
  • sub / super
  • Length or percentage

Visual demonstration

HTML
<p>H<span class="up">2</span>O</p>
CSS
.up { vertical-align: super; font-size: .7em; }

Rendered output

Reference line super · sub · top · middle

💡 Tips

For block layout use Flexbox/Grid alignment instead — vertical-align only works on inline/table-cell boxes.

🌐 Browser notes

Has no effect on block-level elements.

✅ Common use cases

Chemical formulas (H₂O), exponents, aligning icons with text.

typography

direction

Definition. Sets the base writing direction of text (left-to-right or right-to-left).

Syntax

selector { direction: ltr | rtl; }

Accepted values

  • ltr — left-to-right (English, default)
  • rtl — right-to-left (Arabic, Hebrew)

Visual demonstration

HTML
<html lang="ar" dir="rtl">…</html>
CSS
[dir="rtl"] { direction: rtl; }

Rendered output

This sentence flows left to right.

This sentence flows right to left.

💡 Tips

Prefer the HTML dir attribute for accessibility; CSS direction mirrors it.

🌐 Browser notes

Also affects column direction in tables and Flexbox main-axis start.

✅ Common use cases

Multilingual sites, Arabic/Hebrew layouts.

typography

unicode-bidi

Definition. Works with direction to override the default Unicode bidirectional text algorithm for embedded text.

Syntax

selector { unicode-bidi: normal | embed | isolate | bidi-override; }

Accepted values

  • normal
  • embed / bidi-override
  • isolate / isolate-override (recommended)
  • plaintext

Visual demonstration

HTML
<bdo dir="rtl">reversed</bdo>
CSS
.override {
  direction: rtl;
  unicode-bidi: bidi-override;
}

Rendered output

A mixed sentence مرحبا with embedded RTL text.

This text has its character order overridden.

💡 Tips

Use the HTML <bdi> element for unknown-direction user content — it implies isolate.

🌐 Browser notes

An advanced property; rarely needed outside multilingual contexts.

✅ Common use cases

Mixed-direction UIs, usernames of unknown language.

2 · Lists

Markers, position & images for <ul> / <ol>

list

list-style-type

Definition. Sets the marker (bullet or number) shown next to each list item.

Syntax

selector { list-style-type: disc | circle | square | decimal | lower-alpha | none; }

Accepted values

  • Unordered: disc · circle · square
  • Ordered: decimal · lower-alpha · upper-alpha · lower-roman · upper-roman
  • none

Visual demonstration

HTML
<ul class="circle">
  <li>Item</li>
</ul>
CSS
.circle { list-style-type: circle; }

Rendered output

  • Square bullet item one
  • Square bullet item two
  1. Roman numeral one
  2. Roman numeral two
💡 Tips

Use list-style-type: none before building custom navigation or menus.

🌐 Browser notes

Many extended keywords exist (cjk-decimal, georgian, etc.).

✅ Common use cases

Feature lists, navigation menus, numbered steps.

list

list-style-position

Definition. Decides whether the marker sits inside the list item's box or outside it.

Syntax

selector { list-style-position: inside | outside; }

Accepted values

  • outside (default) — marker hangs in the margin
  • inside — marker is part of the text flow

Visual demonstration

HTML
<ul class="in">
  <li>Item</li>
</ul>
CSS
.in { list-style-position: inside; }

Rendered output

  • Marker outside — wrapped lines indent past the bullet.
  • Marker inside — wrapped lines align under the bullet.
💡 Tips

inside is handy when list items have background colors.

🌐 Browser notes

Default is outside.

✅ Common use cases

Styled bullet badges, tight cards.

list

list-style-image

Definition. Replaces the default marker with an image.

Syntax

selector { list-style-image: url("marker.svg") | none; }

Accepted values

  • url("…") — path to the image
  • none — fall back to list-style-type

Visual demonstration

HTML
<ul class="custom">
  <li>Item</li>
</ul>
CSS
.custom { list-style-image: url("star.svg"); }

Rendered output

  • Star marker item one
  • Star marker item two
💡 Tips

Image markers cannot be sized or recolored — for control use ::marker or a background-image hack instead.

🌐 Browser notes

If the image fails, the browser falls back to list-style-type.

✅ Common use cases

Branded bullets, checkmarks, custom icons.

list

list-style (shorthand)

Definition. Sets list-style-type, list-style-position and list-style-image in one declaration.

Syntax

selector { list-style: <type> <position> <image> | none; }

Accepted values

  • Any combination, in any order
  • none resets all three

Visual demonstration

HTML
<ul class="reset">…</ul>
CSS
.reset { list-style: none; }
.sq   { list-style: square inside; }

Rendered output

  • Square inside, item one.
  • Square inside, item two.
  • No marker at all.
  • No marker at all.
💡 Tips

Order of values does not matter — omitted values reset to defaults.

🌐 Browser notes

Shorthand is the recommended way to reset lists for nav.

✅ Common use cases

Navigation resets, one-line list styling.

table

border-collapse

Definition. Decides whether adjacent table cells share a single border or keep their own.

Syntax

selector { border-collapse: collapse | separate; }

Accepted values

  • collapse — one shared border between cells
  • separate (default) — each cell keeps its own border

Visual demonstration

HTML

CSS
.clean { border-collapse: collapse; }

Rendered output

NameRole
AriaDesigner
LeoEngineer
💡 Tips

collapse is the modern default for data tables.

🌐 Browser notes

border-spacing has no effect when collapsed.

✅ Common use cases

Data grids, calendars, comparison tables.

table

border-spacing

Definition. Sets the distance between cell borders (only when border-collapse: separate).

Accepted values

  • One length (uniform)
  • Two lengths (horizontal & vertical)

Visual demonstration

CSS

Rendered output

AB
CD

Use this to create "card grid" tables without margins.

Ignored under border-collapse: collapse.

✅ Common use cases

Spaced data cells, dashboard widgets.

Definition. Controls the algorithm the browser uses to lay out table cells.

Accepted values

  • auto (default) — column widths follow content

4 · Display

How boxes participate in the flow

layout

display

Definition. The most important layout property — it decides the box type an element generates and how it flows.

Syntax

selector { display: block | inline | inline-block | none | contents | flex | grid; }

Accepted values covered here

  • block — full width, new line, respects width/height
  • inline — flows with text, ignores width/height
  • inline-block — flows inline but respects box properties
  • none — removed from the flow entirely
  • contents — the box itself disappears; children join the parent's flow

Visual demonstration

HTML
<div class="blk">block</div>
<span class="ib">inline-block</span>
CSS
.blk { display: block; width: 100%; }
.ib  { display: inline-block; width: 80px; }

Rendered output

I am a block — I take the full width and start on a new line.
inline · inline · inline-block · inline-block
💡 Tips

display:none removes the element from the accessibility tree too. To hide visually but keep it, use visually-hidden techniques.

🌐 Browser notes

contents had accessibility bugs historically — test before relying on it.

✅ Common use cases

Nav bars (inline-block), showing/hiding (none), buttons that sit inline.

5 · Position

static · relative · absolute · fixed · sticky + offsets & z-index

position

Definition. Decides how an element is placed in (or removed from) the normal document flow, and which coordinate system its offsets refer to.

Syntax

selector {
  position: static | relative | absolute | fixed | sticky;
  top: <length>; right: <length>;
  bottom: <length>; left: <length>;
  z-index: <integer>;
}

Accepted values

  • static (default) — normal flow, ignores offsets
  • relative — offset from its normal position; flow kept
  • absolute — removed from flow; positioned vs nearest positioned ancestor
  • fixed — positioned vs the viewport (stays on scroll)
  • sticky — relative until a threshold, then fixed

Visual demonstration

HTML
<div class="anchor">
  <span class="badge">New</span>
</div>
CSS
.anchor { position: relative; }
.badge { position: absolute; top: -8px; right: -8px; }

Rendered output

static — in flow
relative — nudged 10px left & up
absolute — pinned to stage corner
💡 Tips

Always set position: relative on the parent before using absolute on a child.

🌐 Browser notes

sticky needs a scrolling ancestor and a top/bottom threshold.

✅ Common use cases

Tooltips, badges, modals (fixed), sticky headers.

top · right · bottom · left · z-index

Definition. The four offsets move a positioned element along its axes; z-index controls stacking order among positioned siblings.

Syntax

selector {
  top: 10px; right: 0; bottom: auto; left: 50%;
  z-index: 5;
}

Accepted values

  • Offsets: <length> · % · auto
  • z-index: integer (negative allowed) or auto

Visual demonstration (z-index stacking)

HTML
<div class="layer">1</div>
<div class="layer top">2</div>
CSS
.layer { position: absolute; }
.top  { z-index: 10; }

Rendered output

z-index 1
z-index 2
z-index 3
💡 Tips

Offsets only work on non-static elements. z-index also requires a positioned element.

🌐 Browser notes

Each stacking context isolates its children's z-index from the outside.

✅ Common use cases

Modals over content, dropdowns, badges, layered cards.

6 · Float

Wrapping text around elements & the clear property

layout

float & clear

Definition. float pulls an element to one side so that inline content wraps around it; clear pushes an element below floated siblings.

Syntax

selector { float: left | right | none; }
selector { clear: left | right | both | none; }

Accepted values

  • float: left · right · none
  • clear: left · right · both · none

Visual demonstration

HTML
<img class="wrap" src="pic.jpg">
<p>Text wraps around…</p>
<p class="after">Cleared.</p>
CSS
.wrap   { float: left; }
.after  { clear: both; }

Rendered output

F

This paragraph flows around the floated box. Notice how the text reflows naturally on the side opposite the float, exactly like a magazine column wrapping a pull-image.

This element has cleared the float and starts on a new line.
💡 Tips

Floats still work great for wrapping text around images. For whole layouts, prefer Flexbox/Grid.

🌐 Browser notes

A floated element is taken out of normal flow, so its parent may collapse — use a clearfix.

✅ Common use cases

Images inside articles, drop caps, magazine layouts.

7 · Visibility

visible · hidden · opacity — the three ways to "hide" an element

layout

visibility & opacity

Definition. visibility:hidden hides an element but keeps its space; opacity:0 makes it fully transparent (still interactive); display:none removes it entirely.

Syntax

selector {
  visibility: visible | hidden | collapse;
  opacity: 0;   /* 0 to 1 */
}

Accepted values

  • visibility: visible · hidden · collapse
  • opacity: 0–1 (decimals allowed)

Visual demonstration

HTML
<div class="secret">Hidden</div>
<div class="ghost">Faded</div>
CSS
.secret { visibility: hidden; }
.ghost  { opacity: 0; }

Rendered output

visible
hidden
opacity 0
none

Notice "hidden" still occupies space, "opacity 0" keeps space & clicks, "none" leaves no trace.

💡 Tips

For fade-in/out animations use opacity + transition; visibility is not smoothly animatable.

🌐 Browser notes

opacity:0 elements are still focusable & clickable — combine with visibility:hidden to disable them.

✅ Common use cases

Modals, tooltips, image rollovers, print-only content.

cursor

Definition. Sets the mouse pointer shown when hovering over an element — an important cue for how it can be interacted with.

Syntax

selector { cursor: pointer | default | text | move | grab | crosshair | wait | not-allowed | help | zoom-in | zoom-out; }

Accepted values

  • pointer · default · text
  • move · grab · grabbing
  • crosshair · wait · not-allowed · help
  • zoom-in · zoom-out
  • Custom: url("c.cur"), auto

Hover the grid above — each box applies the named cursor. The right cursor signals affordance before the user clicks.

💡 Tips

Use cursor: pointer on clickable non-button elements, not-allowed on disabled controls.

🌐 Browser notes

Exact cursor glyph varies by OS; the keyword meaning stays the same.

✅ Common use cases

Links & buttons (pointer), drag handles (grab), disabled (not-allowed), help icons (help).
